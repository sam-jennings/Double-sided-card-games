## 1. Player Experience

### Core Principles

- **Know your player before you design.** Every design decision flows from understanding who you're designing for. Misaligned audience = misaligned game.
- **Find your core tenant and maximise it.** If your game is hyper-competitive, lean in. If it's about storytelling, ensure those moments are supported. Excel at something specific.
- **You cannot serve incompatible motivations.** Players who want direct conflict and players who hate confrontation cannot both be your primary audience. Choose.

### Player Type Framework

| Type | Primary Motivation | Design For | Example Games |
|------|-------------------|------------|---------------|
| **Family** | Quality time together | Accessibility, short rules, inclusive | Ticket to Ride, Zombie Kids |
| **Social/Party** | Social experience | Light rules, memorable moments, laughter | Codenames, Werewolf, Just One |
| **Strategic** | Deep decision-making | Crunchy systems, optimisation paths | Brass, Terraforming Mars |
| **Thematic** | Story and immersion | Narrative, atmosphere, character | Cthulhu: Death May Die |

Most players are blends (e.g., 75% thematic, 25% strategic). Twilight Imperium is simultaneously thematic, strategic, and social.

### Why People Play

1. **To Win** — Competitive drive, min-maxing, tournament play
2. **To Have Fun** — Escapism, cool moments, joy of play itself (enjoy even when losing)
3. **To Socialise** — Face-to-face connection, shared experience, relationship-building
4. **To Tell a Story** — Shared narrative moments that become group lore

### Engagement Heuristics

- If players argue about strategy → social/interactive game working
- If players optimise silently → strategic game working
- If players retell moments afterward → thematic/narrative game working
- If players immediately want a rematch → engagement is high
- If players check phones mid-game → pacing or engagement failure

### The Magic Circle

Games define spaces where normal social rules don't apply. In Werewolf, lying is required. In competitive games, attacking friends is expected. Design your magic circle intentionally — make clear what's allowed and what's not.

### Accessibility Framework

| Element | Family Games | Strategy Games |
|---------|-------------|----------------|
| **Rulebook** | 1-4 pages max | Extended OK if clear |
| **Card Text** | Large font, minimal | Dense acceptable |
| **Turn Structure** | 1-2 simple steps | Multiple phases OK |
| **Rule Exceptions** | Near zero | Tolerable if justified |
| **Game Length** | 30-60 min | 2-8 hours if engagement holds |

**Golden Rule:** Make every mechanic as intuitive as possible, regardless of audience. Discovered through playtesting with your target players.

### Tactile Experience

Physical components significantly impact game appeal. Azul's wooden tiles > cardboard tokens. Carcassonne's tile-laying appeals to non-gamers as jigsaw puzzle. Component quality is a design decision, not just production.

## 2. Mechanics and Systems

### Core Principles

- **Mechanics are tools in service of your vision, not goals.** You don't pick mechanics from a list — they emerge from the experience you're creating.
- **Design for deep strategy, not complexity for its own sake.** Nobody likes things because they're complex. They like depth despite complexity.
- **Even complex games should be as intuitive as possible.** Always ask: can this be simpler without losing depth?

### Design Approach: Top-Down vs Bottom-Up

| Approach | Start With | Best For | Risk |
|----------|-----------|----------|------|
| **Top-Down** (Theme First) | Theme, emotion, world | Licensed games, emotion-driven design | Mechanics may not support vision |
| **Bottom-Up** (Mechanics First) | Novel mechanic, system | Innovation-driven design | Theme feels pasted on |

Both are valid. Many designers naturally favour one. The key is intentionality.

### Component-Mechanic Relationships

| Component | Strengths | Weaknesses | Best For |
|-----------|-----------|------------|----------|
| **Dice** | Fun to roll, tactile | Swingy, limited info space, expensive custom | Randomness with excitement |
| **Cards** | Flexible, info-rich, predictable over time | Less tactile | Hand management, variable powers |
| **Tokens in bags** | Tactile draw, hidden information | Limited info per token | Resource randomness |
| **Boards** | Spatial relationships, shared state | Expensive, table space | Area control, movement |

### Three Sources of Complexity

1. **Number of subsystems** — How many independent systems interact?
2. **Weight of each mechanic** — How heavy is each individual system?
3. **Fiddliness** — How many different component types and piles to track?

Reduce any one to reduce overall complexity without necessarily losing depth.

### Danger Mechanics (Use With Caution)

| Mechanic | Why It's Risky | Mitigation |
|----------|---------------|------------|
| **Roll to Move** | No player agency | Offer choice (roll 2, pick 1) |
| **Lost Progress** | Feels terrible to lose built-up work | Limit setbacks, offer recovery |
| **Take That** | Singles out players, creates bad feelings | Spread effects, make optional |
| **Player Elimination** | Sitting out is miserable | Only late-game, game ends soon after |
| **Unnatural Flow** | Players constantly forget steps | Redesign if players consistently fight the flow |

**Player elimination CAN work** if: it happens late, the game ends soon after, and players expect it (Cthulhu: Death May Die heroic sacrifice model).

### Heuristics

- If a rule takes a page to explain one card → redesign the card
- If players consistently play wrong → the design is fighting their intuition
- If nobody uses an ability → it's either weak or unclear
- If everyone uses one strategy → it's probably overpowered
- Price point and complexity should align ($100 game needs matching depth)
- Novelty matters but isn't essential — execution can be enough

### Variety and Replayability

All games that stick with players have moments where someone does something unexpected and players react. Even Love Letter with 12 cards creates endless variety because people + randomness = unique situations every time.

## 3. Interaction and Dynamics

### Direct vs Indirect Interaction

| | Direct | Indirect |
|--|--------|----------|
| **Pros** | Agency, politics, drama, memorable moments | Control own destiny, can't be destroyed, fewer negative emotions |
| **Cons** | Personal targeting, ganging up, kingmaking | Multiplayer solitaire, can't stop winner |
| **Best For** | War games, social deduction, confrontational themes | Engine builders, Euro-style, puzzle games |

### Two-Player Game Dynamics

**Strengths:** Intense focus, skill-proof, personal victory.
**Risks:** Snowball effect, skill gap discourages casuals, personal defeat feels harsh.

### Scaling for Player Count

- **Downtime is the #1 multiplayer problem.** As player count rises, time between turns becomes critical.
- **Solutions:** Interlaced turns (everyone does step 1, then step 2), active engagement during others' turns, avoid volatile game state between turns.
- **Start playtesting at 2 players, expand to target count.** Friction points emerge quickly.
- **Publishers want higher player counts** (more buyers). Push back if it compromises experience — poor first impressions at max player count hurt reputation.

### Five Keys to Good Cooperative Games

1. **Spinning Plates** — More threats than time to address them. Constant tension.
2. **Thematic Help** — Players have complementary abilities with limited sharing.
3. **Individual Importance** — Every player gets spotlight moments that save the group.
4. **Team Strategy + Individual Agency** — Discuss together, decide individually.
5. **Difficulty Scaling** — Target ~40% win rate. Offer difficulty modes (avoid "novice" naming).

### Quarterbacking Prevention

The biggest co-op problem: one player solves and commands everyone.

| Solution | How It Works |
|----------|-------------|
| **Hidden Information** | Players can't see each other's hands |
| **Randomness** | Dice/draws make optimal play uncertain |
| **Limited Communication** | Restrict what players can reveal |

### Communication Rule Design

Ask: "If I was trying to break this game, what would I say?" Then create explicit rules showing what IS and ISN'T allowed, with specific examples.

### Solo Gaming Design

- **Co-op games:** Usually work solo with minimal changes. Increase difficulty (complete information makes it easier).
- **Competitive games:** Add scoring targets or automated opponent decks (5-10 cards per enemy type, thematic behaviour).
- **Games that CAN'T go solo:** Highly social games (Werewolf), heavy trading games (Catan).
- **Design tip:** Build co-op games solo first, then expand to multiplayer.

## 4. Theme and Narrative

### Core Principles

- **Theme and narrative are integral, not afterthoughts.** They shape mechanics, writing, and player experience.
- **Theme = setting/world. Narrative = specific story within it.** Related but distinct.
- **When theme and mechanics clash → be flexible with mechanics.** There are many mechanics; find one that supports your theme.

### Theme Representation

| Channel | Function | Example |
|---------|----------|---------|
| **Artwork** | Most visible thematic signal | Cover art, card illustrations |
| **Flavor Text** | In-world immersion without blocking gameplay | Character quotes on cards |
| **Rulebook** | Intro stories, in-world explanations, sidebars | Horus Heresy data pads |
| **Component Design** | Physical objects that feel in-world | Star Wars: Rebellion Tantive IV door cards |
| **Thematic Language** | Renamed game terms (deck → "stack", discard → "archives") | Android: Netrunner |

### Narrative Type Decision

| Type | Control | Replayability | Best For |
|------|---------|---------------|----------|
| **Scripted** | High (designer controls pacing) | Low (players know the story) | Mystery, story-driven campaigns |
| **Emergent** | Low (player-driven) | High (different every time) | Sandbox, replayable adventures |
| **Hybrid** | Medium | Medium-High | Flavor text as scaffolding for player stories |

**Sign of success:** Players post session reports on BGG that sound like they were actually there.

### Thematic Consistency Audit

Ask for every element:
- Do player actions make sense for their character?
- Is the game type (co-op/competitive) aligned with the theme?
- Do victory conditions match the narrative goal?
- Does the game end at a natural climactic moment?
- Does thematic language enhance or confuse?

### Emotional Design Framework

1. **What emotions do I want players to feel?** (Heroic? Scared? Gleeful? Determined?)
2. **Does my writing match those emotions?** (Dark themes → dark writing)
3. **Do my mechanics support those emotions?** (Fear → scary combat)
4. **Is everything reinforcing?** (Theme + writing + mechanics + components aligned)

### Immersion Breakers

| Breaker | Why It Hurts | Fix |
|---------|-------------|-----|
| Bad writing | Players laugh at game, not with it | Hire an editor, get feedback |
| Too much writing | Interrupts gameplay, bores players | Max 2-3 paragraphs before play starts |
| Out-of-place mechanics | Narrative logic breaks | Retheme or replace mechanic |
| Mismatched victory conditions | Winning doesn't feel like the story's conclusion | Align win condition with narrative goal |
| Min-maxing behaviour | Players do narratively absurd things | Use mechanics to prevent, not theme |

## 5. Visual Design and Presentation

### Visual Hierarchy Principles

- **What players need most frequently → highest in hierarchy.** Think like a video game UI.
- **Fewer things emphasised = more they stand out.** Bold everything → nothing stands out.
- **Natural reading order:** Top-left = most important, Bottom-right = least. Bottom-right often hidden when cards fanned in hand.

### Card Layout Framework

| Position | Priority | Notes |
|----------|----------|-------|
| Top-left | Highest | First seen, always visible in hand |
| Top-right | High | Visible in hand but less prominent |
| Centre | Medium | Art/main content area |
| Bottom-left | Low | Often partially hidden |
| Bottom-right | Lowest | Hidden when fanned, obscured by hand |

### Colour and Accessibility

- **Never rely on colour alone.** Always pair with icons, patterns, or text labels.
- Many types of colourblindness exist beyond red-green.
- Test with colourblind simulators (Photoshop, online tools).
- Seven Wonders uses both colours and icons for card types.

### Icon Design Rules

- Distinctive silhouettes (vary shapes — not all circles)
- Must work at small scale (simpler for smaller icons)
- Single-colour functional (works in black, white, any colour)
- Icons reinforce text, don't replace it
- Use icons where text can't fit (dice, tokens)

### Typography

- **8pt minimum**, preferably larger
- If text doesn't fit → redesign the card, don't shrink the font
- Avoid illegible ornate fonts
- Fix widows and orphans
- High contrast between text and background
- Avoid coloured text (localization cost — black plate replacement is cheaper)
- Same fonts and sizes across all cards of a type

### Localization Design

- German words average 12 letters vs English 5 — leave space
- Leave 3-5 blank text lines at card bottom
- Never put text on dice or tokens (prohibitively expensive to localise)
- Black plate replacement: print all cards without text, then print text plate per language

### Component Specifications

| Component | Minimum Size | Notes |
|-----------|-------------|-------|
| Tokens | 18mm diameter | Anything smaller is hard to handle |
| Token thickness | 4-5mm | Less than 4mm feels too thin |
| Card sizes | Poker, bridge, mini (standard) | Custom sizes require special dies (extra cost) |

### Material Tradeoffs

| Material | Cost | Shapes | Feel | Notes |
|----------|------|--------|------|-------|
| Cardboard | Cheapest | Any | Least premium | Default option |
| Wood | Medium-High | Simple/blocky only | Premium, heavy | Printing on wood adds significant cost |
| Plastic | Medium-High | Complex shapes OK | Variable | Requires tooling for injection molds |

### The Guile Principle

Include one standout component that catches attention at conventions and in photos. Players at adjacent tables noticing your components = free marketing. Examples: Game of Thrones sword tokens, Root printed wooden meeples.

## 6. Art Direction and Illustration

### Critical Rules

- **DO NOT use AI art in finished commercial games.** Community backlash, boycotts, reputation damage. OK for early prototyping only.
- **If pitching to publishers: DON'T commission finished art.** Mood board sufficient. Publishers want their own art direction. Save your money.
- **If self-publishing: commission art only after mechanics are solid.** Starting art too early = wasted money when design changes.

### Art Process

1. Get 2-3 initial samples to set style direction
2. Find artist whose portfolio matches your vision
3. Iterate on pencil sketches with feedback
4. Approve style as baseline for all artwork
5. Colour versions created
6. Graphic designer adds design elements on top

### Finding Artists

| Source | Strengths |
|--------|-----------|
| BoardGameGeek credits | Find artists behind games you love |
| ArtStation | Searchable by style, wide selection |
| Social media (Instagram) | Portfolio browsing |
| Game website credits | Direct connection to working game artists |
| OBA Collaborative | Platform for contracts, payments, tracking |

### Working With Artists

- **Collaborative mindset** — creative partners, not servants
- **Don't micromanage** — allowing freedom produces better work
- **Clear descriptions** — highlight gameplay requirements but allow compositional freedom
- **Always add 2-3 months buffer** to deadlines
- **Kill fee contracts** (typically 50%) protect both parties
- **Pay promptly** — artists talk, reputation matters for future availability

### Art Pricing Guide

| Type | Price Range |
|------|------------|
| Premium card art (MtG level) | ~$500/card |
| Standard card art | $200-300/card |
| Simple/no background | $100-150/card |
| Black and white | ~$150 or less/card |
| Box art/cover | $2,000-5,000 |

### Reducing Art Requirements

- Text-heavy cards don't need illustrations
- Generic tokens use simple geometric shapes
- Reuse artwork across duplicate components
- Reserve art budget for key components (characters, cover, bosses)
- Photography works for historical games with proper licensing

### Technical Specs for Commissioning

- 300 DPI for print quality
- 0.25" bleed on all edges
- TIFF or high-quality PDF format
- RGB or CMYK per printer requirements
- Specify dimensions art will be printed at

### Idea Protection Reality

- Ideas are easy; execution is hard. Most publishers have more ideas than capacity.
- Theft is more likely after publication success (counterfeits, knockoffs) than during pitching.
- Bad actors get blacklisted — industry relies on relationships.
- Studios don't steal ideas because reputation damage costs more than the idea's value.

## 7. Rules and Communication

### Core Principles

- **Writing rules reveals design problems.** A rule that takes a page to explain one card = card is too complex.
- **Rulebooks serve two purposes in tension:** Teaching first-time players AND reference during play.
- **Clarity over brevity, but achieve both.** Avoid unneeded words. Use simple language. Break up text walls.

### Rulebook Structure

1. **WHO** — Give players context ("You are settlers in a foreign land...")
2. **WHAT** — Victory condition ("Accumulate 20 victory points")
3. **HOW** — Brief action overview ("Journey, collect resources, vanquish foes")
4. **Game Type** — Competitive, cooperative, or teams (state explicitly)
5. **Components** — Pictures + quantities only (defer function explanations)
6. **Setup** — Numbered steps with large visuals
7. **Gameplay** — Establish round/turn structure BEFORE specifics
8. **Detailed Rules** — Actions, phases, specifics
9. **Clarifications/FAQ** — Edge cases from playtesting
10. **Index** — Alphabetical for complex games

### Point of View Decision

| POV | Pros | Cons | Best For |
|-----|------|------|----------|
| Second person ("You") | Casual, welcoming | Hard with asymmetric roles | Family games |
| Third person ("The player") | Precise, handles multiple roles | Wordy, verbose | Complex/asymmetric games |

Always avoid gendered pronouns.

### Two-Rulebook Solution

For complex games, separate Learn to Play (first game, preset boards) from Rules Reference (alphabetised, complete lookup). The two must not contradict. Teach players explicitly to use Reference during play, not Learn to Play.

### Three Criteria for Good Rulebooks

1. Can you read the first page and get an idea of the experience?
2. Can you quickly look up a card icon during play without someone sighing?
3. Are players more excited about playing after reading?

### Golden Rule for Edge Cases

Include a catch-all: "If a situation is not covered, resolve it in the way that is least beneficial to the player causing the situation." Acknowledges imperfection while keeping play moving.

### Rulebook Timing

- Start writing after first successful playtest (even as placeholder outline)
- Update every couple weeks, not after every playtest
- Fully playable (rough Word doc OK) by external testing phase
- Don't constantly polish during early development — rules will change

### Practical Guidelines

- Font: 10-11pt (12 usually too large, 9 occasionally OK)
- Leave room for localisation (text lengths vary by language)
- Use headers (H1, H2, H3) for visual hierarchy
- Don't bury important info in sidebars — reference from main text
- QR codes to tutorial videos supplement but don't replace written rules
- Errata: wait 2-3 weeks post-launch, post on website + BGG
- Pitch rulebooks: clear enough to learn/play, doesn't need to be pretty

## 8. Playtesting and Iteration

### Core Principles

- **Your first prototype will fail. That's expected and valuable.**
- **Playing reveals what your game actually is.** Theory can't substitute for table experience.
- **Expect 10+ significant prototype iterations** for a publishable game.
- **Don't balance a game nobody wants to play.** Fun first, balance later.

### Prototyping

- Build core mechanic prototype in 1-2 days
- Materials: card sleeves (print + cut + sleeve with backing), generic tokens, various dice, printer, paper cutter
- First prototype needs to be playable, not beautiful
- Push past perfectionism to get to table

### Four Playtesting Stages

| Stage | Purpose | Who | What to Watch |
|-------|---------|-----|---------------|
| **Solo Dry Run** | Verify mechanics work | You alone | Obvious breaks, session length |
| **First Playtest** | Core experience check | Friends/family | Fun? Clear? What breaks? |
| **Local Playtesting** | Iterative refinement | Regular group | Frustrations, confusion, disengagement |
| **External/Blind** | Fresh eyes, standalone test | Strangers, no designer present | Rules clarity, balance, standalone playability |

### Observation Signals

- **Phone-checking** → Pacing/engagement problem
- **Players consistently play wrong** → Unintuitive design
- **One strategy always wins** → Balance issue
- **Nobody uses an ability** → Weak or unclear
- **Players argue about rules** → Ambiguous writing
- **Immediate rematch requests** → High engagement

### Feedback Framework

**Always ask:**
1. What was your favourite part?
2. What was your least favourite part?
3. What confused you?
4. Did you have fun? Why or why not?

**Explicitly request honest negative feedback** — people default to politeness.

### When to Ignore Feedback

- **Contradictory feedback** between groups (less actionable than consistent feedback)
- **Late-stage feedback on finished elements** (set expectations about what you're testing)
- **Solution-oriented feedback** — ask for the root problem, then design your own solution

### Feedback Triage

When players complain about a mechanic: is the problem **clarity** or **design**?
- Try better explanation first (rulebook, teach, card text)
- Only make mechanical changes if clarification doesn't solve it
- This prevents reactive changes that chip away at intentional design

### Balance Testing

- Comes during external playtesting phase, NOT early
- Track: wins/losses, overpowered/underpowered elements, victory path frequency
- Ask players to flag cards that feel too strong or too weak
- Dig deeper: is the card actually weak, or just unclear?

### Designer Blindness

After 12+ months of playtesting, designers become blind to issues. You need fresh external eyes. The relationship between designer feedback and player feedback is symbiotic — each sees what the other can't.

## 9. Production and Manufacturing

### Manufacturing Paths

| Path | Volume | Best For | Key Consideration |
|------|--------|----------|-------------------|
| **Large Factory** (China/Europe) | 1,000+ copies | Commercial release | Long timeline, economies of scale |
| **Print-on-Demand** (Game Crafter) | <500 copies | Testing, proof of concept | Higher per-unit, less quality control |

### Major Manufacturers

- **LudoFact** (Germany) — Well-known German manufacturer
- **Panda** (China) — ~90% of Kickstarter games, most in-demand, potentially most expensive
- **Longshore, LongPack** (China) — Alternatives to Panda

Always get quotes from 2-3+ factories AND request samples to compare quality vs cost.

### Manufacturing Timeline

| Phase | Timing | Notes |
|-------|--------|-------|
| Quoting | 6+ months before print | Cost conversations early; may redesign components |
| Material Samples | 3+ months before | White sample (unprinted game to feel quality) |
| File Submission & Proofing | 2-3 weeks | Digital proofs (PDFs) + Physical proofs (ALWAYS request) |
| Pre-Production | 2-4 weeks | Printing plates, die creation |
| Printing & Assembly | 10-16 weeks | Longer with plastic/complex components |
| Shipping | 2-3 months | Factory → port → ocean → dock → warehouse |

**Total from file submission to receiving games: 5-9 months.** Then 3-4 more weeks to reach customers.

### Component Cost Hierarchy (Cheapest → Most Expensive)

Cardboard tokens → Punch board standees → Standard dice → Wood pieces ≈ Plastic pieces → Acrylic standees → Custom figures (sculpting + tooling required)

### Quality Control Checkpoints

1. **White Sample** — Blank unprinted game. Feel the quality in your hands.
2. **Digital Proofs** — PDFs to verify files transferred correctly.
3. **Physical Proofs** — Printed samples. ALWAYS request, especially first game. Colours vary from screen.
4. **Pre-Production Sample** — Proper materials, properly cut.
5. **Mass Production Sample** — Final confirmation before shipping.

### Critical Warnings

- **Boxes are hardest to fix** — be very careful with box files. Shipping empty boxes costs as much as full games.
- **Wrong barcodes** require stickering every copy in the warehouse.
- **Custom inserts** usually get cut during budgeting. Gameplay > packaging.
- **Reprints** skip proofing/pre-production but still take 4-6 months.
- **No full board game manufacturing in the US** currently.
- **Kickstarter:** Don't finalise print quantity until campaign ends.

### Safety and Compliance

- EU, US, UK each have different standards
- Large factories handle certification testing (adds cost)
- Young age ratings have size restrictions, lead paint testing, hazardous materials testing

## 10. Business and Pricing

### The Times Five Model

**Production cost × 5 = MSRP.** This is the baseline for sustainable margins.

| MSRP | Production Cost Target | Notes |
|------|----------------------|-------|
| $20 | $4/unit | Simple card game |
| $50 | $10/unit | Standard board game |
| $100 | $20/unit | Premium/deluxe game |

**Without shipping factored in, use ×7 or ×8 instead.**

### Supply Chain Breakdown ($50 MSRP Example)

| Entity | Pays | Receives |
|--------|------|----------|
| Factory | — | ~$10 (production cost) |
| Publisher | ~$10 | ~$20 (sells to distributor) |
| Distributor | ~$20 | ~$35 (sells to store) |
| Game Store | ~$35 | ~$50 (sells to customer) |

### Cost Types

| Type | Behaviour | Examples |
|------|-----------|---------|
| **Fixed** (per print run) | Same per unit regardless | Components, assembly, shipping |
| **One-Time** (pay once) | Divided across print run | Artwork, graphic design, tooling |

One-time cost impact: $10,000 art ÷ 1,000 copies = $10/unit. Same art ÷ 5,000 copies = $2/unit. **Print run size dramatically affects per-unit cost.**

### Crowdfunding Pricing Trap

Don't use ×2 for crowdfunding then ×5 for retail. This creates massive price gaps (Gloomhaven: $79 KS → $200 retail). Apply ×5 consistently, offer max 10% discount for backers.

### Print Run Decision

| Channel | Realistic First Run | Notes |
|---------|-------------------|-------|
| Kickstarter (indie) | 1,000-2,000 copies | Most successful KS ~1,000. Top 1% = 5,000+ |
| Traditional publisher | 5,000+ copies | They have marketing reach |

**Overprinting risk:** Unsold inventory is dead money. Sometimes selling out > overprinting.

### Designer Income Reality

- Most games sell <5,000 copies/year
- Royalties: typically 4-6% of wholesale price
- Royalty advance: $5,000-$10,000 on signing (against future royalties)
- Build livable income by stacking royalties across multiple published games
- Rare blockbusters (Wingspan) can support solo, but that's the exception

## 11. Marketing and Engagement

### Discovery Channels (Ranked)

1. Influencers (YouTube/TikTok)
2. Friends (word-of-mouth)
3. Social media
4. Board Game Geek

### Marketing Timeline

| Phase | Timing | Key Activities |
|-------|--------|---------------|
| **Pre-Launch** | 3-4 months before | Announcement, designer journals, rulebook release, influencer previews, ads, conventions |
| **Launch** | Release week | Full press, review copies ship |
| **Post-Launch** | Ongoing | Free content, reviews, paid content, tournaments, continued ads |

**Cadence:** Post substantive content weekly or biweekly during pre-launch.

### Working With Influencers

- **Reviews should NEVER be paid.** Free copy only. If they ask for payment to review → red flag.
- **Previews, playthroughs, tutorials ARE paid.**
- Match influencer voice to your game's tone
- Be friendly, don't pressure for coverage (<50% return rate on review copies)
- Diversify: YouTube, podcasts, TikTok, blogs, comics
- Bigger channels = busier (your game may get buried). Smaller channels = more accessible.

### Brand Building

- Always be positive — one bad interaction can ruin a lifetime reputation
- People research you online and find past comments
- Professional behaviour matters more than professional credentials

### Convention Value

Conventions offer what online can't: personal interactions, hands-on play, word-of-mouth through friend networks. Most have indie/unpublished game areas. Start with small local cons if budget is tight.

### Do Something Unique

Include at least one distinctive marketing element. Examples: interactive website, create-your-own-character tool, choose-your-own-adventure experience. But don't over-commit to ongoing obligations.

## 12. Pitching and Selling

### The Elevator Pitch Test

Can someone relay your game to others after hearing your pitch? (Telephone game test.) If the description survives two retellings accurately, your pitch is working.

### Pitch Sheet Framework

A one-page game CV with three functions: **Inform**, **Excite**, **Estimate Costs**.

**Required elements:**
1. **Game Overview** — Theme + gameplay (narrative promise + mechanical delivery)
2. **Component List** — Specific ranges ("50-60 action cards"), not vague
3. **Big Three Stats** — Players (range), Ages (range), Time (range)
4. **Unique Selling Proposition** — What makes THIS game special vs all others in the category

### Three Questions Every Pitch Must Answer

1. **Who are you?** (Player's role in the game)
2. **What do you do?** (Core actions during gameplay)
3. **How do you win?** (Victory condition)

### The ABCs

- **Accuracy** — Truthful, factual. Don't exaggerate.
- **Brevity** — Fewer words = easier to relay = survives telephone game
- **Clarity** — No jargon. Accessible to unfamiliar audiences.

### Pitch Presentation Skills

**What works:** Clear positioning, genuine enthusiasm, structured delivery, balanced mechanical depth with accessibility, market awareness.

**What fails:** Too much mechanical detail too soon, vague positioning, lack of passion, ignoring market, over-committing to specific numbers that are estimates.

### Publisher Expectations

| What to Bring | What NOT to Bring |
|--------------|-------------------|
| Functional prototype | Finished artwork |
| Mood board | Commitment to specific art direction |
| Vision document | Rigid price commitments |
| Clear component list | Licensed IP (unless publisher relationship exists) |
| Passion and clarity | Defensiveness |

### Finding Publishers

- **Cardboard Edison** — Best resource. Directory updated yearly. ~$2/month.
- Research publishers whose catalog matches your game
- Conventions (Gen Con, UK Games Expo) have unpublished game areas
- SAZ and TTGDA provide contract templates and guidance

### What Publishers Look For

1. Clarity of vision
2. Realistic assessment
3. Market awareness
4. Professional presentation
5. Passion

**Designer credentials matter less than the game on the table.**

## 13. Game Design Toolkit

### Game Design Checklist

Use this step-by-step when starting a new game:

- [ ] **Define target player type** (Family / Social / Strategic / Thematic / blend)
- [ ] **Define the core experience** — What emotion should players feel?
- [ ] **Write aspirational moments** — Specific moments you want at the table
- [ ] **Create vision document** — Pillars that must be preserved
- [ ] **Choose design approach** — Top-down (theme first) or Bottom-up (mechanics first)
- [ ] **Define victory condition** — Aligned with theme and narrative
- [ ] **Define core loop** — What players do on a typical turn
- [ ] **Build first prototype** (1-2 days, not beautiful)
- [ ] **Solo dry run** — Verify mechanics work
- [ ] **First playtest** — Expect something to break
- [ ] **Iterate** (10+ prototypes expected)
- [ ] **Start writing rulebook** — Even as placeholder outline
- [ ] **External/blind playtesting** — Rules must stand alone
- [ ] **Balance testing** — Only after core is fun
- [ ] **Define component list** with cost awareness
- [ ] **Create mood board** — Visual direction, not commitment
- [ ] **Build pitch sheet** — One page, three functions
- [ ] **Create mock budget** — Fixed + one-time costs, ×5 model

### Design Questions to Ask Yourself

**Player Experience:**
- Why would players replay this?
- What is the emotional payoff?
- What will players tell their friends about?
- Does the game end at its natural climax?

**Mechanics:**
- Are all these rules necessary?
- Can this be simpler without losing depth?
- Do my mechanics emerge from the experience I want, or am I forcing them?
- If I removed this mechanic, would anyone miss it?

**Theme:**
- Do player actions make sense for their character?
- Does the victory condition match the narrative?
- Is the game type (co-op/competitive) aligned with the theme?
- What emotions does my writing evoke?

**Accessibility:**
- Can a new player learn this from the rulebook alone?
- Is the font readable? The icons clear?
- Did I rely on colour alone anywhere?
- Does the turn structure feel natural?

**Business:**
- What is my target MSRP? Does my component list support it?
- Which publishers match this game's style?
- What is my unique selling proposition vs the top 3 competitors?

### Tradeoff Matrix

| Tradeoff | Lean Left When... | Lean Right When... |
|----------|-------------------|-------------------|
| **Complexity vs Accessibility** | Targeting strategic gamers | Targeting family/social gamers |
| **Depth vs Speed** | Players want long sessions | Players want quick plays |
| **Randomness vs Control** | Theme demands chaos/excitement | Players want skill to dominate |
| **Direct vs Indirect Interaction** | Theme is confrontational | Theme is building/puzzle |
| **Scripted vs Emergent Narrative** | Mystery/campaign game | Replayable sandbox |
| **Premium vs Affordable Components** | Competitive differentiation needed | Price sensitivity high |
| **Novel Mechanics vs Familiar Mechanics** | Differentiation is core USP | Execution/theme is the draw |
| **Thematic Language vs Mechanical Clarity** | Immersion is critical | Accessibility is critical |

### Rapid Evaluation Framework

Score any game concept 1-5 on each:

| Criterion | Question | Score |
|-----------|----------|-------|
| **Clarity** | Can I explain this in 30 seconds? | /5 |
| **Engagement** | Why would players come back? | /5 |
| **Differentiation** | What's unique vs existing games? | /5 |
| **Feasibility** | Can this be manufactured at my target price? | /5 |
| **Emotional Hook** | What feeling does this create? | /5 |

15+ = Strong concept. 10-14 = Needs work. <10 = Rethink.

### Prototype Iteration Loop

```
DESIGN → BUILD (1-2 days) → SOLO TEST → PLAYTEST → OBSERVE → ANALYSE → REFINE → REPEAT
```

**Key signals per iteration:**
- Iteration 1-3: Is the core fun? What breaks?
- Iteration 4-6: Are mechanics serving the experience? What's unnecessary?
- Iteration 7-9: Is it balanced? Does it stand alone?
- Iteration 10+: Polish, edge cases, blind testing

### File Management System

- Version individual components (game_board_v4, cards_v6) — not entire game
- Keep ALL old files — reference past iterations when stuck
- Maintain dated playtest notes: bold issues, strikethrough when fixed, never delete
- Don't constantly update rulebook during early development — only when external testers need it
